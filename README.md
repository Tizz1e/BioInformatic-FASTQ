# BioInformatic-FASTQ

В ходе решение был реализован простой парсер FASTQ и несколько вспомогательных функций:
- GetStatistics - возвращает статистику по файлу
  - Общее число записей
  - Минимальная длина записи
  - Максимальная длина записи
  - Средняя длина записи
- GCContent - возвращает GCContent в виде процента
- Trim - производит тримминг записей методом скользящего окна

# Ход решения задач
1. Просто выводим статистику по файлу
2. Используем функцию GCContent
3. Используя парсер, пробежались по записям и честно посчитали среднее значение качества
4. Комбинируем Trim и GetStatistics, чтобы ответить на поставленные вопросы

# Встретившиеся проблемы
По сравнению с остальными задачами, реализовать тримминг оказалось очень сложно. Проблема была не в алгоритме, а просто в не совпадении результатов работы моей реализации и Trimmomatic, хотя в какой-то момент я полностью повторил их реализацию. Решение оказалось простым - изменить значение quality с 30 до 31, после чего результат полностью совпал. Не понятно, почему такое произошло, ведь алгоритмы идентичны с точностью до знаков сравнения.
